# v1.1 框架接入与模块边界

## 核心层速览

- **Core**：主管理器 MainManager、ManagerUtility 等基础服务，负责启动链、模块注册、生命周期协调。
- **Event**：全局事件系统，处理事件订阅、发布与异步传播，供各模块之间传播状态。
- **FSM**：有限状态机框架，用于表达流程与流程节点之间的跳转逻辑，与 Procedure、StepSystem 配合。
- **Procedure**：业务流程（Procedure）入口，按顺序调度 UI、Assets、Hotfix、Audio 等模块。
- **Assets**：资源主链路（LWAssets），通过 ManagerUtility.AssetsMgr 暴露统一接口，承担资源加载、卸载、更新。
- **UI**：界面与 View 层，依赖 Assets/Hotfix 提供 UI 资源并驱动视图生命周期。
- **Hotfix**：热更集成层，提供 ByCode/Reflection 两种模式、HybridCLR 支持，并可以被 Procedure 或 Runtime 控制。

## 默认热更路线

- **ByCode**：默认热更模式，宿主加载 C# 代码到流程中，依托 Procedure + Hotfix 实现业务入口的初始化与跳转。
- **Reflection（可选）**：在需要反射加载补丁时，Framework 通过 Assets 模块从固定目录加载 DLL（详见下文）。

## Reflection 加载约定

Reflection 模式的 DLL 通过 Assets 模块按需加载，必须将编译后的程序集拷贝到 `Assets/Hotfix/` 目录，框架启动链会在 Hotfix 层统一扫描并注册到 HybridCLR/ILRuntime（或自定义逻辑）。宿主需要在 Startup 中配置需要加载的程序集列表，Assets 负责实际的文件加载与缓存，确保 `Hotfix` 层获得正确字节。

## 宿主项目责任

- 担任 **Startup/Bootstrap** 的实现，控制 Core → Assets → UI → Procedure → Hotfix 等链路的启动顺序并在 ManagerUtility 中完成注册。
- 负责撰写业务 **Procedure**，定义流程节点、状态跳转以及携带 UI/Assets 资源的加载顺序。
- 编写业务 **UI/View** 层，确保界面通过 ManagerUtility.AssetsMgr 或 Hotfix 资源路径获取依赖，并与 Procedure 解耦。
- 提供场景、Prefab、配置表、素材、贴图、Audio、Shader 等项目资源（包括 `Assets/0Res` 等目录）供 LWAssets 管理。
- 决策是否启用 **Audio** 模块以及音频资源的目录/加载节奏，Audio 由宿主配置资源路径与播放策略。
- 决策是否启用 **StepSystem** 插件：StepSystem 属于业务插件层，仅在宿主显式开启时挂载到流程，默认情况下不加入核心启动链。

## StepSystem 插件层说明

StepSystem 不在核心启动链之内，它属于可选的业务插件层。宿主在需要时显式启用 StepSystem，使其在 Procedure 中按需创建/驱动流程节点。StepSystem 可以依赖 Core、Event、Assets、UI 等核心模块，但核心层不能反向依赖 StepSystem，以避免循环引用与生命周期冲突。StepSystem 的配置与资源都由宿主管理，框架只提供 Step 节点执行与资源绑定的基础能力。

## 最小接入步骤

1. **引入框架**：将 LWFramework 的核心包（含 Core、Assets、Hotfix、UI、Event、FSM 等）添加到宿主工程，确保 `Assets/LWFramework/RunTime` 及依赖资源同步。
2. **准备 Startup**：实现 `Startup`/`Bootstrap`，在 `MainManager` 中注册 Core、Assets、Event、Hotfix 等管理器，让 `ManagerUtility.AssetsMgr` 可用。
3. **配置首个 Procedure**：定义业务入口流程，关联 UI View 与 Hotfix 逻辑；Framework 会在流程中调用 Assets+UI 来完成资源加载与界面绘制。
4. **默认使用 ByCode**：默认的热更路线为 ByCode，只需保证 Procedure 与 Hotfix 代码存在即可自动执行，无需额外资源。
5. **Reflection 选项**：若要启用 Reflection 热更，在 Startup 中配置要加载的程序集名称，并把对应 DLL/bytes 放入 `Assets/Hotfix/`，Assets 模块会自动读取并交给 HybridCLR。
6. **按需开启 Audio**：宿主决定是否引入 Audio 模块，配置播放逻辑与资源目录后交由 Assets 管理加载。
7. **显式启用 StepSystem**：若需要 StepSystem，宿主需在初始化阶段手动注册 StepSystem 管理器与步骤定义，并在 Procedure 中触发，不自动加入默认启动链。
